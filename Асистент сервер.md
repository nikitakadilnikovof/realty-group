# Асистент сервер

## Что это

Сервер AI-ассистента работает через Cloudflare Worker. Статичный сайт отправляет запросы на Worker, а Worker уже безопасно обращается к OpenAI и Telegram. Ключи и токены не попадают в браузер.

Адрес Worker:

```text
https://realty-ai-assistant.nikitakadilnikovof.workers.dev/
```

Полный актуальный код Worker лежит в файле:

```text
cloudflare-worker-ai-assistant.js
```

## Что умеет Worker

У Worker есть два режима работы.

1. Обычный диалог с ассистентом.

Клиент отправляет сообщение пользователя, историю переписки, язык, текущую страницу и ID объекта. Worker загружает базу объектов, выбирает релевантные варианты и отправляет в OpenAI только короткий список подходящих объектов, а не всю базу.

2. Заявка на звонок.

Когда пользователь нажимает кнопку `Заказать звонок` и вводит телефон, клиент отправляет на тот же Worker запрос с:

```json
{
  "action": "callback_request",
  "phone": "телефон пользователя",
  "history": "история переписки",
  "language": "ru",
  "page": "текущая страница",
  "url": "текущий URL",
  "propertyId": "id объекта"
}
```

Worker формирует сообщение для менеджера и отправляет его в Telegram через бота.

## Что отправляет сайт для обычного ответа

Клиентский файл `ai-assistant.js` отправляет на сервер JSON:

```json
{
  "message": "сообщение пользователя",
  "history": "последние сообщения текущего чата",
  "language": "ru",
  "page": "текущая страница",
  "url": "текущий URL",
  "propertyId": "id объекта, если пользователь находится на странице объекта"
}
```

История переписки хранится в браузере в `localStorage`. На сервер отправляются последние сообщения текущего сеанса, чтобы OpenAI мог отвечать с учетом контекста.

## Как сервер работает с базой объектов

Worker загружает базу объектов из переменной окружения `PROPERTIES_URL`.

Пример:

```text
https://raw.githubusercontent.com/nikitakadilnikovof/realty-group/main/data/properties.json
```

Важно использовать именно `raw.githubusercontent.com`, а не ссылку GitHub вида `/blob/`, потому что `/blob/` возвращает HTML-страницу, а не JSON.

После загрузки базы Worker:

1. Считает релевантность каждого объекта по сообщению пользователя.
2. Оставляет только объекты с совпадениями.
3. Сортирует по релевантности.
4. Берет максимум 8 объектов.
5. Отправляет в OpenAI только сокращенную версию этих объектов.

Если пользователь задает общий вопрос про документы, налоги, ВНЖ или закон и в тексте нет совпадений с объектами, OpenAI не получает случайные объекты.

## Что возвращает сервер

Для обычного диалога Worker возвращает:

```json
{
  "answer": "ответ ассистента",
  "responseId": "id ответа OpenAI",
  "matchedProperties": [
    {
      "id": "id объекта",
      "objectNumber": "номер объекта",
      "title": "название объекта"
    }
  ]
}
```

Клиент использует `matchedProperties`, чтобы показать превью объектов и сделать ссылки кликабельными внутри чата.

Для заявки на звонок Worker возвращает:

```json
{
  "ok": true
}
```

## Настройки Cloudflare

Нужны секреты:

```text
OPENAI_API_KEY
TELEGRAM_BOT_TOKEN
TELEGRAM_CHAT_ID
```

Нужна переменная:

```text
PROPERTIES_URL
```

`OPENAI_API_KEY` используется для OpenAI. `TELEGRAM_BOT_TOKEN` и `TELEGRAM_CHAT_ID` используются только для отправки заявки менеджеру в Telegram.

## Что увидит менеджер в Telegram

Менеджеру придет сообщение примерно такого вида:

```text
Новая заявка из AI-ассистента

Телефон / WhatsApp: +90...
Язык: ru
Страница: property
URL: https://...
ID объекта: tr-kemer...

История переписки:
Клиент: ...
Ассистент: ...
Клиент: Заказать звонок: +90...
```

Так менеджер сразу видит, о чем человек спрашивал, какие объекты обсуждались и с какой страницы пришла заявка.
